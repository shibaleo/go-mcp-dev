## コンテキスト

MCPist管理UIの構築にあたり、デザイン発案から実装までのワークフローと技術選定を検討する。

### 要件

1. **管理UI機能**
   - トークン管理（Long-lived Token発行・失効）
   - サービス連携（OAuth連携の追加・解除）
   - Tool Sieve設定（ロール・権限管理）
   - ユーザー設定

2. **技術制約**
   - Next.js使用が決定済み
   - Supabase Authとの統合

3. **開発体制**
   - 個人開発（専任デザイナーなし）
   - 高速なプロトタイピングが必要

## 選択肢

### Option A: Figma → 手動コーディング（従来型）

```
Figma（デザイン）→ 手動コーディング → Next.js
```

| 評価項目 | 評価 |
|---------|------|
| デザイン品質 | ◎ ピクセルパーフェクト可能 |
| 開発速度 | × 数日〜数週間 |
| デザイナー必要性 | × 必要（またはFigmaスキル） |
| 個人開発向け | × 不向き |

**適用シーン**: デザイナーがいるチーム、ブランドガイドラインが厳密

### Option B: v0 + shadcn/ui（高速プロトタイプ型）

```
プロンプト → v0 → Next.js/shadcn/uiコード → 調整
```

| 評価項目 | 評価 |
|---------|------|
| デザイン品質 | ○ 十分な品質（shadcn/uiベース） |
| 開発速度 | ◎ 数時間〜数日 |
| デザイナー必要性 | ◎ 不要 |
| 個人開発向け | ◎ 最適 |

**適用シーン**: 個人開発、MVP、高速プロトタイピング

### Option C: shadcn/ui直接利用

```
shadcn/uiコンポーネント → 組み合わせ → Next.js
```

| 評価項目 | 評価 |
|---------|------|
| デザイン品質 | ○ 統一感あり |
| 開発速度 | ○ 中程度 |
| デザイナー必要性 | ○ 不要（センスは必要） |
| 個人開発向け | ○ 適している |

**適用シーン**: UIパターンが明確、デザインセンスに自信あり

### Option D: Figma + v0ハイブリッド

```
Figma（ワイヤーフレーム）→ v0（コード化）→ 調整
```

| 評価項目 | 評価 |
|---------|------|
| デザイン品質 | ◎ 事前に設計可能 |
| 開発速度 | ○ 中程度 |
| デザイナー必要性 | △ 基本的なFigmaスキルは必要 |
| 個人開発向け | ○ 適している |

**適用シーン**: ある程度設計を固めてから実装したい場合

## 各ツール詳細

### v0 (Vercel)

**概要**: プロンプトからReact/Next.jsコードを生成するAIツール

**特徴**:
- shadcn/ui + Tailwind CSSベースのコード生成
- プレビュー付きで即座に確認可能
- 生成コードはそのままNext.jsプロジェクトに統合可能

**使い方**:
```
1. https://v0.dev にアクセス
2. プロンプト入力（例: 「ダッシュボード with サイドバー, トークン管理テーブル」）
3. 生成されたコードをコピー
4. Next.jsプロジェクトに統合
5. 必要に応じて調整
```

**料金**: 無料枠あり、有料プランで生成回数増加

### shadcn/ui

**概要**: Radix UI + Tailwind CSSベースのコンポーネントコレクション

**特徴**:
- コピペで使える（node_modulesに依存しない）
- カスタマイズ容易
- アクセシビリティ対応（Radix UIベース）
- Next.js App Router対応

**インストール**:
```bash
npx shadcn-ui@latest init
npx shadcn-ui@latest add button card table dialog
```

**主要コンポーネント**:
| コンポーネント | 用途 |
|--------------|------|
| Button | ボタン |
| Card | カード |
| Table | テーブル（トークン一覧等） |
| Dialog | モーダル |
| Form | フォーム（設定画面等） |
| Tabs | タブ切り替え |
| Sidebar | サイドバーナビゲーション |

### Figma

**概要**: クラウドベースのデザインツール

**特徴**:
- ブラウザで動作
- リアルタイムコラボレーション
- プロトタイプ機能
- デザインシステム構築可能

**用途（MCPistの場合）**:
- 画面構成の検討（ワイヤーフレーム）
- デザインの事前確認
- 複雑な画面のレイアウト設計

**料金**: 無料プランあり

### Storybook

**概要**: UIコンポーネントの開発・ドキュメント化ツール

**特徴**:
- コンポーネント単体での動作確認
- ビジュアルリグレッションテスト
- ドキュメント自動生成

**用途（MCPistの場合）**:
- 初期段階では不要
- コンポーネントが増えてきたら導入検討

## MCPist管理UIの画面構成案

### 必要な画面

| 画面 | 機能 | 優先度 |
|------|------|:------:|
| ダッシュボード | 概要表示、クイックアクション | 中 |
| トークン管理 | Long-lived Token発行・一覧・失効 | 高 |
| サービス連携 | OAuth連携サービス一覧・追加・解除 | 高 |
| Tool Sieve設定 | ロール・権限・ツールフィルタ設定 | 中 |
| ユーザー設定 | プロファイル、通知設定 | 低 |

### 画面イメージ（v0プロンプト例）

**トークン管理画面**:
```
MCPサーバーのトークン管理画面。
- ヘッダーに「API Tokens」タイトル
- 「新規トークン発行」ボタン
- トークン一覧テーブル（名前、作成日、最終使用日、ステータス、アクション）
- 各行に「失効」ボタン
- shadcn/ui、ダークモード対応
```

**サービス連携画面**:
```
MCPサーバーのサービス連携画面。
- 連携済みサービスをカード形式で表示
- 各カードにサービスアイコン、名前、連携日、「解除」ボタン
- 「新規連携」ボタンでサービス選択モーダル
- 対応サービス: Google Calendar, Notion, GitHub, Zaim, freee
- shadcn/ui、ダークモード対応
```

## 検討事項

### 1. デザインシステムの必要性

| 規模 | デザインシステム | 推奨 |
|------|----------------|------|
| MVP段階 | 不要 | shadcn/uiデフォルトで十分 |
| 成長段階 | 検討 | カラーパレット、タイポグラフィ統一 |
| 拡大段階 | 必要 | Figmaでデザインシステム構築 |

### 2. ダークモード対応

shadcn/ui + Tailwind CSSで標準対応可能:
```tsx
// next-themes使用
<ThemeProvider attribute="class" defaultTheme="system">
  {children}
</ThemeProvider>
```

### 3. レスポンシブ対応

管理UIの場合:
- デスクトップ優先（主な利用環境）
- タブレット対応（サブ）
- モバイル対応（最低限）

### 4. 既存実装（dwhbi）の流用

dwhbiに管理UI実装があれば流用可能:
- 認証フロー（Supabase Auth）
- トークン管理画面
- サービス連携フロー

## 追加ツール評価

以下のツールについて、MCPist管理UIでの適用可否を検討した。

### カテゴリ別整理

| カテゴリ | ツール | MCPist適用 |
|---------|--------|:----------:|
| **AIプロトタイプ生成** | Uizard | △ |
| | Google Stitch | × |
| | Framer AI | × |
| **AIデザイン支援** | Figma AI | △ |
| | Canva AI | × |
| **画像生成** | Midjourney | × |
| **UX分析** | Attention Insight | △ |
| | Hotjar AI | ○ |

### 各ツール詳細

#### Uizard

**概要**: AIでワイヤーフレーム・プロトタイプを自動生成

**特徴**:
- スクリーンショット・手書きスケッチからUI生成
- テーマ・スタイル適用機能
- デザインファイル出力（コード出力なし）

**MCPist評価**: △ 不採用
- 理由: v0がReact/Next.jsコードを直接生成するため、デザインファイル出力のUizardは不要
- 用途: デザイナーとの協業時、または詳細なワイヤーフレームが必要な場合のみ

#### Google Stitch

**概要**: GoogleのAI UIデザインツール（2025年発表）

**特徴**:
- プロンプトからUIデザイン生成
- HTML + Tailwind CSS出力
- Figmaプラグイン連携

**MCPist評価**: × 不採用
- 理由: React/Next.jsコード出力ではなくHTML出力。v0の方がNext.js統合に適している
- 用途: 静的サイト、またはReact不要なプロジェクト向け

#### Framer AI

**概要**: AIウェブサイトビルダー + Wireframer機能

**特徴**:
- プロンプトからウェブサイト生成
- Wireframer: 高速ワイヤーフレーム生成
- Framerプラットフォーム上でホスティング

**MCPist評価**: × 不採用
- 理由: Framerプラットフォーム依存。Next.js + Supabase構成には統合不可
- 用途: Framer上で完結するランディングページ等

#### Figma AI (Figma Make)

**概要**: Figma内蔵のAIデザイン支援機能

**特徴**:
- First Draft: プロンプトからデザイン生成
- AIによる画像編集・背景除去
- デザインシステム自動適用

**MCPist評価**: △ 将来検討
- 理由: 現段階ではFigma自体を使用しない方針。デザインシステム構築段階で再検討
- 用途: デザインシステムが必要になった成長段階で検討

#### Canva AI (Magic Studio)

**概要**: Canva内蔵のAIデザイン機能

**特徴**:
- Magic Design: プロンプトからデザイン生成
- Magic Edit: AI画像編集
- プレゼン・SNS投稿向け

**MCPist評価**: × 不採用
- 理由: マーケティング素材向け。UIコンポーネント生成には不向き
- 用途: ランディングページのヒーロー画像、プロモーション素材

#### Midjourney

**概要**: AI画像生成ツール

**特徴**:
- 高品質なアート・イメージ生成
- スタイル指定可能
- Discord経由で利用

**MCPist評価**: × 不採用
- 理由: UIコンポーネント生成ではなく画像生成ツール。管理UIには不要
- 用途: ブランドイメージ作成、カスタムイラスト・アイコンが必要な場合

#### Attention Insight

**概要**: AIヒートマップ・UX分析ツール

**特徴**:
- デザインのアイ・トラッキング予測
- クリック予測ヒートマップ
- ユーザーテスト前の検証

**MCPist評価**: △ 将来検討
- 理由: MVP段階では不要。ユーザー数増加後、UX改善フェーズで有用
- 用途: 重要な画面（オンボーディング、料金ページ等）の最適化

#### Hotjar AI

**概要**: UX分析プラットフォーム + AI機能

**特徴**:
- ヒートマップ・セッション録画
- AI Surveys: インテリジェントなアンケート
- AI分析レポート生成

**MCPist評価**: ○ 将来導入予定
- 理由: 実ユーザーの行動分析に有用。ローンチ後のUX改善に必須
- 用途: ユーザー行動分析、離脱ポイント特定、フィードバック収集

### 評価まとめ

```
┌─────────────────────────────────────────────────────────────────┐
│                    ツール適用タイムライン                         │

│                                                                 │
```

## 暫定決定

**Option B: v0 + shadcn/ui** を採用予定。

### 理由

1. **開発速度**: 個人開発で高速プロトタイピングが必要
2. **品質**: shadcn/uiベースで十分な品質
3. **学習コスト**: Figma習得より即座に実装開始可能
4. **柔軟性**: 生成コードは完全にカスタマイズ可能

### 実装ステップ

```
1. v0でプロンプト入力、主要画面のコード生成
2. Next.jsプロジェクトに統合
3. Supabase Auth連携
4. 必要に応じてコンポーネント調整
5. dwhbi実装があれば流用
```
