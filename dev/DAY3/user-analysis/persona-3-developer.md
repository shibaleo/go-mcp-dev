# 開発者ペルソナ分析（最終版）

## 1. ペルソナ概要

- 立場：非IT業種に勤める個人開発者
- 開発時間：業務外のプライベート時間
- 技術背景：
  - 情報系学部・資格なし
  - 独学で Jekyll + Netlify によるサイト構築経験あり
  - Claude Code を活用し、短時間で高品質なコードを生成できる
  - MCP発表から1週間で自作MCPサーバーを開発・クラウドホストする実行力
- 開発動機：
  - 自分自身の課題を、自分のサービスで解決することが最大の快感
  - 課題が解決された瞬間に「成長」を実感する
  - 技術そのものより「アイデアが形になること」に価値を感じる


## 2. 成功の定義（What is “Success”）

この開発が成功している状態とは、次の3点が同時に成立していること。

1. **自分の思考がすっきりしている**
   - 何をすべきかが明確
   - 不要な判断が減り、集中できる
   - 思考のノイズが減っている

2. **現実の業務・作業が確実に楽になっている**
   - 単純作業が意識せず流れる
   - 待ち時間や手戻りが減少
   - タスクや状況が構造（DAGなど）として可視化されている

3. **同じ課題を持つ少人数が実際に便利さを享受している**
   - 多くのユーザーは不要
   - テスト利用・フィードバックが得られる関係性
   - 改善の手応えがあること


## 3. 開発スタイル

### Issueドリブン

- 抽象的な理想設計から入らない
- 明確な「困りごと」から着手
- まず動かし、後から整える
- 完璧さよりも「価値が出ること」を優先

### AIとの関係性

- AIは伴走者・作業者・調査担当
- 抽象設計や最終判断は人間が担う
- 並列調査・コード生成・下準備はAIに任せる
- コミュニケーションに関わる判断は必ず人間が行う


## 4. 許容できる失敗／できない失敗

### 許容できる失敗

- ローカル・個人環境での破損
- 後から修復可能なデータ消失
- 一時的に動作しなくなること
- 雑でも後から直せる実装

### 絶対に避けたい失敗

- セキュリティ事故
- 他人のデータ侵害
- 信頼関係を壊すコミュニケーションミス
- 自分の実装を起点とした情報漏洩


## 5. 技術選定に現れる価値観

- ローカル完結・個人PC内で完結
- 長期間放置しても壊れにくい構成
- 障害対応が頻発しない
- ログは必ず残す（学習・分析対象）
- バックグラウンドで処理できる非同期設計
- 長期の自分利用は許容、他人のための長期運用はしない


## 6. 公開・利用方針

- OSSとしてコード公開は許容
- 同じ課題を持つ少人数に使ってもらえたら成功
- SaaS化・創業・大規模運用は想定しない
- 自分の実装が評価されることは純粋に嬉しい


## 7. 得意・関心領域

- AI OCR
- 数式処理
- RAG（Retrieval Augmented Generation）
- Knowledge Graph
- Web Scraping
- 論文検索


## 8. 開発における優先順位

1. 動くことを確認
2. テストコードをAI生成・実行
3. セキュリティチェックを別セッションで実施
4. ここまで到達したら内部実装は深追いしない


## 9. このペルソナが示す設計原則（示唆）

- 人間は「判断と責任」に集中する
- AIは「雑音を取り除く」
- 思考を妨げる待ち時間・往復を極力減らす
- 小さく作り、一生使える核を残す
- 「便利になった」と実感できる瞬間を最優先する


## 10. ペルソナを一文で表すと

> **「思考を澄ませるためにAIを使い、
> 自分と同じ課題を持つ誰かを静かに楽にする個人開発者」**

## 関連ストーリー

- PSN-3-US-01: 思考を妨げない開発体験がほしい
- PSN-3-US-02: まず動くものを素早く作りたい
- PSN-3-US-03: テストとセキュリティはAIに支援してほしい
- PSN-3-US-04: セキュリティ事故だけは絶対に避けたい
- PSN-3-US-05: ローカル環境で壊れることは許容したい
- PSN-3-US-06: 長期間放置しても動き続けてほしい
- PSN-3-US-07: ログを学習・分析対象として残したい
- PSN-3-US-08: 自分の課題を自分のサービスで解決したい
- PSN-3-US-09: 同じ課題を持つ少人数に使ってもらいたい
- PSN-3-US-10: 判断と責任は自分で持ちたい
