# ペルソナ・ストーリー・受け入れ基準 サマリー

## 概要

1人のユーザーが3つの帽子（ペルソナ）を被り分けて利用する。
同じ人だが、文脈によって関心事・使うサービス・期待するUXが異なる。

## 共通の特性

3つのペルソナに共通する特性:

- 技術に精通している
- 複数のサービス・LLMを利用している
- Context Rotを認識している
- ベンダーロックインを避けたい
- 自分のデータは自分で管理したい
